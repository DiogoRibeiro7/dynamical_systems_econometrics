API reference

The package is organized around contracts and config, simulation, rare-event diagnostics, recurrence analysis, multivariate stress, empirical data loading and transforms, econometric baselines, and plotting/experiment orchestration. The import root is dynsys_econometrics.

Contracts and config

  • TimeSeriesFrame: validated long-format panel contract.
  • EventFrame: validated threshold-event contract.
  • ExperimentResult: registry of saved experiment outputs.
  • load_config: load YAML experiment settings.
  • resolve_output_dir: determine reproducible output paths.

Simulation

  • logistic_map: iterate the logistic map from an initial state.
  • simulate_logistic_map: return a long-format simulation frame.
  • simulate_iid_gaussian: generate iid Gaussian benchmarks.
  • simulate_student_t: generate heavy-tailed iid benchmarks.
  • simulate_ar1: simulate dependent Gaussian autoregression.
  • simulate_garch11: simulate clustered volatility.
  • simulate_pomeau_manneville: simulate intermittent dynamics.
  • simulate_intermittent_map: return an intermittent benchmark frame.
  • simulate_coupled_logistic_maps: build a multivariate chaotic example.

Extremes

  • extract_threshold_exceedances: recover exceedance values and timing.
  • block_maxima: compute maxima over fixed windows.
  • peaks_over_threshold: expose POT exceedance magnitudes.
  • runs_declustering: cluster nearby exceedances under a runs rule.
  • runs_declustering_from_indicator: apply the same logic to Boolean exceedance flags.
  • estimate_cluster_size_extremal_index: infer the extremal index from mean cluster size.
  • estimate_runs_extremal_index: estimate the extremal index from runs declustering.
  • estimate_extremal_index: general entry point for extremal-index estimation.
  • threshold_sensitivity_analysis: compare diagnostics across threshold choices.
  • threshold_exceedances: annotate long-format panels with exceedance flags.
  • assign_runs: assign run identifiers to event tables.
  • runs_extremal_index: summarize clustering directly from event tables.
  • mean_excess_table: compute threshold-by-threshold mean excess summaries.

Return times

  • first_hitting_time: find the first threshold crossing after a starting index.
  • compute_return_times: compute recurrence gaps between exceedances.
  • return_time_distribution: summarize recurrence statistics.
  • empirical_survival_curve: estimate the survival function of return times.
  • exponential_benchmark_comparison: compare recurrence data against an exponential benchmark.
  • ks_exponential_diagnostic: quantify deviation from the exponential law.

Recurrence tables

  • return_times: return tidy inter-event durations.
  • inter_event_durations: compute event-level durations explicitly.
  • recurrence_rate: compute rolling or calendar event counts.
  • survival_curve: estimate survival tables from duration data.
  • compare_return_time_distributions: summarize recurrence by regime label.

Multivariate stress

  • wide_panel: pivot long-format series into a wide matrix.
  • joint_exceedances: build a joint-stress event table by date.
  • tail_dependence_coefficient: estimate empirical upper-tail dependence.
  • stress_state_index: aggregate exceedance matrices into a stress score.

Data loading

  • load_timeseries_csv: load a canonical long-format CSV into a validated contract.
  • load_panel_from_directory: combine several local long-format CSV files.
  • load_time_series_csv: normalize a generic local time-series CSV.
  • load_fred_csv: standardize FRED CSV exports.
  • load_fred: fetch FRED data into the standard schema.
  • load_ecb_sdw_csv: normalize ECB Statistical Data Warehouse exports.
  • load_oecd_csv: normalize OECD CSV exports.
  • load_world_bank_csv: normalize World Bank indicator exports.
  • load_yfinance_series: fetch market proxy data through yfinance.
  • write_processed_panel: write validated panels to disk.
  • validate_catalog: validate the public-data catalog metadata.
  • DataLoadFailure: explicit error type for broken source formats.

Transforms and baselines

  • log_returns: grouped log-return transform for positive series.
  • pct_change: grouped percentage changes.
  • rolling_zscore: trailing grouped z-scores.
  • standardize_panel: within-series standardization.
  • winsorize_series: grouped winsorization by quantile.
  • autocorrelation_summary: baseline autocorrelation table.
  • rolling_volatility: trailing volatility diagnostic.
  • drawdown_series: drawdown path by series.
  • regime_summary: descriptive aggregation by regime label.
  • compare_event_diagnostics: align event outputs with baseline diagnostics.

Plotting

  • plot_series_with_threshold: show an observable with its exceedance threshold.
  • plot_orbit_and_observable: compare state evolution and observed series.
  • plot_threshold_exceedances: mark rare events in time.
  • plot_clustered_vs_isolated_extremes: visualize dependence in exceedance clusters.
  • plot_return_time_distribution: display recurrence summaries.
  • plot_extremal_index_by_threshold: inspect threshold sensitivity visually.
  • plot_extremal_index_bars: compare clustering across series.
  • plot_joint_stress_timeline: display multivariate stress through time.
  • plot_baseline_comparison: compare baseline diagnostics with event markers.
  • plot_conceptual_pipeline: render the article-facing workflow diagram.
  • plot_macro_financial_timeline: communicate empirical stress episodes.
  • plot_multivariate_stress_heatmap: visualize synchronized stress across variables.

Experiments and utilities

  • run_synthetic_experiment: execute a synthetic workflow from config.
  • run_empirical_experiment: execute an empirical workflow from config.
  • save_experiment_result: persist tables, figures, and JSON summaries.
  • repo_root: locate the repository root.
  • ensure_directory: create directories deterministically.
  • write_json: persist machine-readable run metadata.

Result objects

  • ExceedanceExtractionResult: threshold exceedance values, indices, and counts.
  • ExtremalIndexResult: extremal-index estimate and supporting diagnostics.
  • ThresholdSensitivityResult: threshold grid outputs for stability checks.
  • ReturnTimeDistribution: summary statistics for recurrence gaps.
  • ExponentialComparisonResult: benchmark comparison outputs for return times.
