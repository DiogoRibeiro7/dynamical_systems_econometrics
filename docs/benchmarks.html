Benchmarks

Benchmarks in this repository are meant to compare workflows and estimators under controlled conditions. They are useful for checking consistency, sensitivity, and failure modes, but they are not a license to overstate out-of-sample economic meaning.

Useful benchmark targets

  • synthetic systems with known structure and fixed seeds
  • comparisons between recurrence diagnostics and simpler baseline summaries
  • threshold sensitivity across the same underlying series
  • output stability under small configuration changes

Commands that typically generate benchmark-style artifacts

python scripts/run_smoke_experiment.py
python scripts/run_synthetic_experiment.py
python scripts/generate_article_figures.py

What benchmark comparisons can support

  • whether one implementation is internally consistent with another diagnostic view
  • whether a method reacts plausibly to stronger clustering or heavier tails in synthetic settings
  • whether the package writes stable result bundles for downstream review

What benchmark comparisons cannot support by themselves

  • claims of empirical superiority on real macro-financial datasets
  • causal conclusions about crisis generation mechanisms
  • publication-grade evidence from a single synthetic scenario

Interpretation rule

Treat benchmarks as controlled checks on behavior, not as a substitute for careful empirical validation. If a benchmark result is interesting, the next step is usually stronger data provenance, uncertainty treatment, and threshold robustness analysis.
