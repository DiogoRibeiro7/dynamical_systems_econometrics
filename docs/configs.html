Config guide

The repository uses small YAML files to separate experiment settings from code. The main split is between synthetic and empirical runs, with separate output directories and optional figure-specific settings.

Config files in the repository

  • configs/default.yaml: baseline project, analysis, plot, and output defaults
  • configs/synthetic.yaml: synthetic experiment example
  • configs/empirical.yaml: empirical experiment example
  • configs/figures.yaml: article-figure configuration surface
  • configs/experiment.yaml: earlier repository experiment stub retained for compatibility

Common structure

experiment:
  name: synthetic_logistic_vs_ar1
  mode: synthetic

analysis:
  threshold_quantile: 0.95
  run_length: 3

outputs:
  directory: outputs/synthetic_logistic_vs_ar1

Synthetic config sections

  • experiment.name: run label saved into the output summary
  • experiment.mode: should be synthetic
  • synthetic.systems: list of benchmark systems such as logistic, AR(1), or GARCH
  • analysis.threshold_quantile: event threshold definition
  • analysis.run_length: declustering gap for the runs estimator
  • outputs.directory: output folder for tables, figures, and JSON summary

Empirical config sections

  • experiment.mode: should be empirical
  • empirical.input_path: local canonical CSV in long format
  • analysis.*: threshold and recurrence settings shared with synthetic runs
  • outputs.directory: empirical run output folder

Figure and project defaults

  • project.seed: reproducibility seed for configurable workflows
  • project.output_dir: fallback output directory
  • plots.dpi: figure resolution setting
  • plots.save_format: default saved figure extension

Useful commands

python -m dynsys_econometrics synthetic --config configs/synthetic.yaml
python -m dynsys_econometrics empirical --config configs/empirical.yaml
python scripts/run_experiment.py --config configs/synthetic.yaml

Interpretation rule

Config files make workflows reproducible, but they do not turn synthetic data into empirical evidence. Always distinguish the configured run mode from the claims you make about the outputs.
