Glossary

This glossary keeps the public docs consistent. The terms below are used repeatedly across the repository and should be read in the same way each time, especially when moving between methods, validation, and research-interpretation pages.

Core terms

Article-facing artifact
A named output, usually a figure or source CSV, intended to support drafting and review of the article materials.
Benchmark
A controlled comparison used to inspect behavior, sensitivity, or implementation consistency rather than to prove empirical superiority.
Contract
An explicit expectation about columns, types, uniqueness, or required fields for inputs and outputs.
Empirical pipeline
A workflow that depends on configured public or external datasets rather than purely synthetic inputs.
Extremal index
A summary quantity used to describe the clustering of exceedances rather than only their marginal frequency.
Fallback synthetic data
A substitute input used to keep workflows runnable when real data is unavailable; useful operationally, but not empirical evidence.
Guardrail
A documented limit on what may be claimed or inferred from the repository’s current outputs.
Recurrence diagnostic
A method or summary based on revisit structure, return times, or related dynamical-systems-inspired behavior.
Stable artifact
An output or interface that the repository intentionally supports and expects downstream users to rely on.
Threshold sensitivity
The extent to which a result changes when the exceedance cutoff or event definition is adjusted.
