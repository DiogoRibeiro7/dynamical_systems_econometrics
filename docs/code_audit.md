# Code Audit

## Current repository structure

- `src/dynsys_econometrics/`: package code
- `scripts/`: smoke, figure, experiment, and catalog entry points
- `configs/`: experiment and figure YAML files
- `tests/`: unit and smoke tests
- `notebooks/`: research notebooks
- `article/`: article draft and generated assets
- `docs/`: methodological and engineering documentation

## Existing implemented functionality

- univariate simulation benchmarks
- threshold exceedances and runs declustering
- return-time diagnostics and exponential benchmark comparison
- public-data loaders for canonical long-format panels
- article figure generation
- notebook workflows

## Missing functionality against the original target architecture

- no causal-design layer
- no heavy model-fitting stack beyond lightweight baselines
- no package CLI under `python -m dynsys_econometrics`
- notebooks do not exactly match the prompt-target filenames

## Risk register

- short empirical samples can make tail diagnostics unstable
- empirical inputs depend on researcher-supplied public data
- multivariate stress diagnostics can be overinterpreted if treated as causal
- optional online sources can change schemas

## Refactor plan by phase

1. contracts and configuration
2. panel transforms and catalog validation
3. multivariate and recurrence layers
4. experiment orchestration
5. documentation and reproducibility hardening

## Test gaps

- no explicit plot smoke tests for the new figure-returning functions
- no direct notebook execution tests

## Data pipeline gaps

- the online fetch layer is catalog validation only
- article-grade empirical panels still depend on locally supplied public data

## Notebook gaps

- current notebooks cover the intended research flow, but not the exact filenames named in the prompt set

## Reproducibility gaps

- GitHub Pages must be enabled in repository settings if the docs site should publish

## Caution

Economic data in this repository is not assumed to be generated by a deterministic map. The code uses dynamical-systems-inspired diagnostics for observed, noisy, policy-sensitive time series.
