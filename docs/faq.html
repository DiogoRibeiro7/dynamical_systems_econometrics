Frequently asked questions

Is this a library, a paper companion, or a workflow repository?

It is all three, but the package layer is the anchor. The repository exposes reusable code, article-facing artifacts, and documented command paths so the research workflow stays inspectable.

Can I run it without external data access?

Yes. The synthetic and smoke workflows are intended to run without live downloads. External public data matters for empirical workflows, not for basic package setup or most validation tasks.

What should I run first?

pip install -e ".[dev]"
python -m dynsys_econometrics --help
python scripts/run_smoke_experiment.py

Where do the main outputs go?

Generated outputs are routed through the configured results and article figure locations. The command map, results page, and article guide explain which artifacts are stable deliverables versus exploratory diagnostics.

Do passing tests validate the empirical claims?

No. Passing tests improve internal reliability. They do not by themselves justify macro-financial interpretation, threshold choices, or causal language.

Can I reuse figures or workflows in my own work?

Yes, subject to license and citation requirements. Reusing software or figure-generation structure is different from reusing substantive conclusions without independent support.

Where should I look if something seems inconsistent?

  • Troubleshooting for common failure paths
  • Validation for quality-check scope
  • Guardrails for interpretation boundaries
  • Workflow for end-to-end command structure
