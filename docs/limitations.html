Limitations

The repository is designed to be useful and reproducible, but it is not a guarantee of empirical truth. Several constraints remain structural: short samples, threshold sensitivity, model simplifications, and the gap between controlled simulations and real macro-financial dynamics.

Method limitations

  • extreme-value summaries can be unstable in short or noisy samples
  • recurrence diagnostics can be informative without uniquely identifying mechanisms
  • baseline econometric comparisons are descriptive unless identification is explicitly justified
  • single-threshold views can hide important sensitivity across cutoff choices

Data limitations

  • public macro-financial sources may be revised, incomplete, or uneven across series
  • normalization improves comparability but does not fix source-quality problems
  • fallback synthetic data keeps workflows runnable but cannot validate empirical claims

Workflow limitations

  • passing tests shows internal consistency, not substantive correctness of every interpretation
  • stable article artifacts help traceability but do not make an argument automatically persuasive
  • thin scripts reduce duplication, yet orchestration can still evolve as the project matures

Interpretation limit

The strongest claims this repository can support are bounded by data provenance, threshold robustness, uncertainty treatment, and the explicit guardrails documented elsewhere. When any of those are weak, the correct action is usually to narrow the claim rather than decorate it with more plots.
