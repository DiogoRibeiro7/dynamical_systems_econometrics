Governance

Governance in this repository is mainly about decision quality: keeping methods, claims, and public documentation aligned while resisting scope creep that would outrun reproducibility and evidence.

Primary governance concerns

  • protecting stable interfaces and documented workflows
  • keeping research claims narrower than the evidence when necessary
  • preferring tested package logic over notebook-only convenience code
  • making scope expansions explicit rather than accidental

Decision standard

  • changes should improve correctness, clarity, or reproducibility
  • new complexity needs a defensible maintenance story
  • public-facing claims need matching provenance, limitation, and results context

Review expectation

Good review should ask not only whether code runs, but whether the repository becomes easier to trust. That includes interfaces, docs, output naming, and the discipline of not overstating what the methods establish.

Scope rule

When a proposed addition increases interpretive risk faster than it increases reliability, it should be narrowed, delayed, or rejected. Governance here is intentionally conservative because research repos fail more often through weak claims than through missing features.
