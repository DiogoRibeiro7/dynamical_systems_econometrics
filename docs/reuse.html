Reuse guide

This repository is designed for software reuse, workflow reuse, and methodological inspection. Those are not the same thing as reusing the repository's substantive interpretations or presenting its exploratory outputs as settled evidence.

Software reuse

  • reuse the package code, configs, and scripts under the repository license terms
  • preserve attribution and keep repository metadata, including citation details, intact
  • when adapting workflows, document which defaults you changed and why

Citation expectation

If you reuse the package, article-facing figures, or methodological framing in technical or academic work, cite the repository metadata in CITATION.cff and reference the exact version or commit whenever possible.

What can be reused safely

  • data contracts and normalized schema conventions
  • simulation and diagnostic workflows
  • figure-generation patterns and output traceability structure
  • testing and validation discipline around numerical edge cases

What needs independent justification

  • empirical claims based on your own data or thresholds
  • policy or macro-financial interpretations derived from exploratory plots
  • any conclusion that extends beyond the repository's documented guardrails
