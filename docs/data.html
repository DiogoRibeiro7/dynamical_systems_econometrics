Data guide

The repository is designed around an offline-first ingestion model with optional public online sources. Core tests should work without internet access, while empirical workflows can be pointed at local canonical CSV files or validated source catalogs.

Canonical schema

date,series_id,value,country,frequency,unit,source

Only date, series_id, and value are required. Optional metadata columns are preserved when present.

Supported public-source families

  • FRED / ALFRED
  • OECD CSV exports
  • World Bank indicator exports
  • ECB Statistical Data Warehouse
  • Yahoo Finance via yfinance

Offline-first workflow

  • Store local source files under a project-controlled directory such as data/raw/.
  • Normalize them into the canonical long format.
  • Validate or describe source metadata through data/catalog.example.yaml.
  • Feed the canonical panel into empirical notebooks or experiment configs.

Useful package entry points

  • load_timeseries_csv
  • load_panel_from_directory
  • load_time_series_csv
  • load_fred / load_fred_csv
  • load_oecd_csv
  • load_world_bank_csv
  • load_ecb_sdw_csv
  • load_yfinance_series
  • validate_catalog
  • write_processed_panel

Catalog validation

python -m dynsys_econometrics validate-catalog --catalog data/catalog.example.yaml
python scripts/fetch_public_data.py --catalog data/catalog.example.yaml

Interpretation rule

Supported source families and loader contracts do not by themselves imply that a dataset has been used in the article. Only saved outputs from a specific empirical workflow should be treated as evidence.
