Notebook guide

The repository currently includes two notebook tracks. One is aligned with the implementation-prompt roadmap. The other is the original research notebook set. Both should call package code rather than embedding reusable analysis logic directly in notebook cells.

Prompt-aligned notebook track

00_method_smoke_test.ipynb

Minimal synthetic workflow with exceedances, extremal index, and return times.

01_synthetic_dynamics.ipynb

Compare logistic-map, AR(1), and GARCH benchmark behavior.

02_empirical_data_audit.ipynb

Audit a canonical empirical panel or synthetic fallback without making substantive claims.

03_rare_events_and_return_times.ipynb

Apply exceedance and recurrence diagnostics to one empirical or fallback series.

04_multivariate_stress.ipynb

Build a joint-stress timeline and stress-state index from multivariate signals.

05_econometric_baselines.ipynb

Compare event diagnostics with baseline volatility-style summaries.

Output convention

The prompt-aligned notebooks save outputs under outputs/notebooks/<notebook_name>/. The article-facing figures should still be treated as script-first assets generated by python scripts/generate_article_figures.py.

Interpretation rule

Empirical notebooks should frame outputs as descriptive diagnostics of recurrence, clustering, and joint stress. They should not imply causal identification or deterministic-law claims about observed economies.
