Command map

This page collects the package CLI and script entry points in one place, together with their output locations and whether they require external data.

Package CLI

python -m dynsys_econometrics smoke

Runs the minimal rare-event comparison between a logistic-map observable and an AR(1) baseline.

External data: no.

Outputs: terminal summary only.

python -m dynsys_econometrics synthetic --config ...

Runs a synthetic experiment from YAML configuration.

External data: no.

Outputs: tables, figures, and run_summary.json under the configured output directory.

python -m dynsys_econometrics empirical --config ...

Runs an empirical experiment on a local canonical CSV panel.

External data: yes, local input path required.

Outputs: tables, figures, and run_summary.json under the configured output directory.

python -m dynsys_econometrics validate-catalog --catalog ...

Validates the public-data catalog metadata.

External data: no.

Outputs: terminal validation summary only.

Script entry points

python scripts/run_smoke_experiment.py

Legacy script version of the minimal smoke workflow.

External data: no.

Outputs: terminal summary only.

python scripts/run_experiment.py --config configs/synthetic.yaml

Generic experiment wrapper for synthetic or empirical modes.

External data: depends on config mode.

Outputs: configured experiment directory.

python scripts/generate_article_figures.py

Generates the article figure bundle and matching source CSV files.

External data: no, but uses data/raw/macro_stress_example.csv if present.

Outputs: article/figures/.

python scripts/fetch_public_data.py --catalog data/catalog.example.yaml

Catalog-validation helper for the offline-first ingestion layer.

External data: no.

Outputs: terminal validation summary only.

pytest -q

mypy

Operational notes

  • The package CLI is the preferred public entry point.
  • Empirical commands assume the canonical long format: date, series_id, value.
  • Article figures are reproducible from script entry points rather than notebook-only execution.
  • Synthetic fallbacks are useful for smoke tests and examples, but they are not empirical evidence.
