Reproducibility

This page summarizes how to install, run, and interpret the repository reproducibly. The repository is designed to work without paid data, hidden credentials, or private machine paths.

Installation

python -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"

Core run commands

python -m dynsys_econometrics smoke
python -m dynsys_econometrics synthetic --config configs/synthetic.yaml
python -m dynsys_econometrics empirical --config configs/empirical.yaml
python scripts/generate_article_figures.py

Quality commands

pytest -q
ruff check .
mypy

Output directories

  • outputs/synthetic_logistic_vs_ar1/: synthetic experiment tables, figures, and run summary
  • outputs/empirical_macro_stress/: empirical experiment tables, figures, and run summary
  • outputs/notebooks/<notebook_name>/: prompt-aligned notebook outputs
  • article/figures/: article-facing figures and matching source CSVs

Data requirements

  • Smoke and synthetic workflows require no external data.
  • Empirical workflows require a local canonical long-format CSV: date, series_id, value.
  • The article-figure script can use data/raw/macro_stress_example.csv if present, but falls back to synthetic data otherwise.

Determinism and seeds

The smoke script, synthetic experiment paths, and article figure generation use fixed seeds where stochastic components are involved. Exact floating-point values can still vary slightly across dependency versions, but the workflows are deterministic at the code level.

Known limitations

  • Synthetic fallbacks are useful for smoke tests and examples, not for empirical claims.
  • Rare-event and recurrence diagnostics are descriptive rather than causal.
  • Short empirical samples can make threshold-based diagnostics unstable.
  • The repository draws methodological inspiration from dynamical systems without claiming that observed economies are literal deterministic maps.
