End-to-end workflow

This repository is designed as a research workflow rather than a standalone estimator. The path is: construct benchmark systems, define rare events, measure recurrence and clustering, then repeat the same logic on macro-financial data.

1. Install the project

python -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"

2. Confirm the package CLI works

python -m dynsys_econometrics
python -m dynsys_econometrics smoke

3. Start from simulated benchmarks

Use the simulation module to compare iid, dependent, intermittent, and multivariate systems before moving to empirical data. The relevant entry points are simulate_iid_gaussian, simulate_student_t, simulate_ar1, simulate_garch11, simulate_pomeau_manneville, and simulate_coupled_logistic_maps.

Notebooks: 00_method_smoke_test.ipynb and 01_synthetic_dynamics.ipynb

4. Define the rare-event region

Choose a high threshold and extract exceedances. Then test how stable the conclusions are under nearby threshold choices.

  • extract_threshold_exceedances
  • block_maxima
  • peaks_over_threshold
  • threshold_sensitivity_analysis

5. Measure dependence in extremes

Rare events in this project are not assumed to be isolated. The key question is whether exceedances arrive in bursts and how much those bursts reduce the effective frequency of independent crises.

  • runs_declustering
  • estimate_runs_extremal_index
  • estimate_cluster_size_extremal_index
  • estimate_extremal_index

Notebook: 03_rare_events_and_return_times.ipynb

6. Analyze recurrence and return times

Once exceedances are defined, use recurrence gaps to test whether the empirical behavior is close to an exponential benchmark or exhibits stronger persistence.

  • first_hitting_time
  • compute_return_times
  • return_time_distribution
  • return_times
  • inter_event_durations
  • empirical_survival_curve
  • survival_curve
  • recurrence_rate
  • exponential_benchmark_comparison
  • ks_exponential_diagnostic

7. Move to empirical macro-financial data

All loaders normalize inputs to the same schema: date, series_id, value. That makes the extreme-event and recurrence tooling independent of the source.

  • load_fred or load_fred_csv
  • load_oecd_csv
  • load_world_bank_csv
  • load_ecb_sdw_csv
  • load_yfinance_series
  • load_panel_from_directory
  • validate_catalog

Notebook: 02_empirical_data_audit.ipynb

8. Extend to joint stress events

When the research question is about coordinated instability across inflation, unemployment, spreads, yields, and markets, work from the multivariate simulation and plotting tools.

Notebook: 04_multivariate_stress.ipynb

9. Compare against baseline diagnostics

Baseline econometric summaries remain useful. The repository includes grouped transforms and simple baselines so recurrence outputs can be aligned with volatility- and drawdown-style diagnostics.

  • log_returns
  • pct_change
  • rolling_zscore
  • rolling_volatility
  • drawdown_series
  • compare_event_diagnostics

Notebook: 05_econometric_baselines.ipynb

10. Produce publication figures

Figures should be reproducible outside the notebooks. The notebooks are exploratory interfaces, while the script is the publication path. 

python scripts/generate_article_figures.py

Notebook: 05_article_figures.ipynb

11. Preserve reproducibility

The expected quality gate is the same one used in CI: 

pytest -q
ruff check .
mypy
