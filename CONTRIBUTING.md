# Contributing

Thanks for your interest in improving this project.

## Ground rules

- Keep changes focused and easy to review.
- Match the existing coding style (PEP 8 and clear, explicit names).
- Add/adjust tests for behavioral changes where practical.
- Update documentation when behavior changes.

## Development workflow

1. Create a descriptive branch from `main`.
2. Implement your changes with clear commit messages.
3. Run relevant checks locally if possible (tests, formatting, linting).
4. Open a PR with a concise summary and rationale.

## Pull request checklist

- [ ] I added tests for new behavior (or explained why none were added).
- [ ] I updated docs where user-facing behavior changed.
- [ ] I kept scope focused and avoided unrelated refactors.
- [ ] I linked any related issue.

## Reporting issues

Please include:

- Reproduction steps.
- Expected vs observed behavior.
- Environment details (OS, Python version, dependencies).
- A minimal example when possible.
